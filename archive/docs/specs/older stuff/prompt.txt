Write a product specification in @specs/product-spec-braids-phases.md for converting @eurorack/braids/ into a highly faithful software instrument built with JUCE.

Key considerations:
	•	The software version will use MIDI input rather than CV, so make thoughtful adaptations where appropriate and use as much of the original code for inspiration.
	•	After defining the product specification, outline development phases and organize them into clear, discrete steps, including audio functionality.
	•	The goal is to deliver a highly faithful feature set.
	•	The original hardware’s rotary encoder (turn + tap) and LED screen interactions must be accurately reproduced in the software version.
    •	The original hardware was monophonic, but since this will be a software synthesizer, it should support polyphony. If you identify other opportunities to expand it now that it's a software synth consider proposing as future enhancements.
    •	Features such as Meta mode must remain fully functional, and the LED display should behave in a way that faithfully mirrors the original firmware.
